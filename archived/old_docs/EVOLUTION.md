# AI 自我進化系統使用指南

## 🧬 什麼是自我進化？

AI 自我進化系統讓你的模型能夠：
1. **記錄每次對話**的質量和用戶反饋
2. **從錯誤中學習**（用戶糾正）
3. **自動生成新訓練數據**（從高質量對話）
4. **定期微調進化**（用新數據重新訓練）

| `/stats` | 查看進化統計（反饋數量、滿意度等） |
| `/evolve` | 生成新訓練數據 |
| `/analyze` | 分析 AI 的弱點 |

| `/quit` | 退出 |

### 反饋選項

| 選項 | 說明 | 效果 |
|------|------|------|
| 1 (👍 很好) | 5星好評 | 此對話將成為高質量訓練樣本 |
| 2 (👌 還可以) | 3星評價 | 記錄但不用於訓練 |
| 3 (👎 不好) | 負面反饋 | 標記為需改進，分析弱點 |
| 4 (✏️ 糾正) | 提供正確答案 | **最重要**！AI 會學習正確答案 |

## 💡 最佳實踐

### 1. **糾正錯誤最重要**

當 AI 回答錯誤時：
```
選擇 (1-4 或 Enter): 4
正確答案是: 我無法預測未來的事情
✅ 感謝糾正！AI 會記住這個
```

這會創建一個**糾正訓練樣本**，AI 下次會學會正確回答。

### 2. **標記好的回答**

遇到特別好的回答時，給 5 星：
```
選擇 (1-4 或 Enter): 1
```

這會確保這種高質量回答被保留和強化。

## 🔄 持續進化週期

建議的進化週期：

1. **階段一（Week 1-2）**
   - 每天使用 30-50 次
   - 積極提供反饋
   - 特別注意糾正錯誤

2. **階段二（Week 3）**
   - 達到 100+ 反饋時進行第一次進化
   - 測試進化效果
   - 對比前後差異

3. **階段三（Week 4+）**
   - 繼續使用進化版本
   - 每月進化一次
   - 保持模型與你的需求同步

## 🛡️ 注意事項

### ✅ 推薦做法
- 糾正明顯錯誤
- 對好的回答給予正面反饋
- 定期檢查統計和弱點
- 每次進化後測試效果

### ❌ 避免做法
- 不要對所有回答都給好評（會降低質量）
- 不要用太少數據進化（< 20 樣本）
- 不要過度訓練（epochs 不要超過 10）
- 不要跳過測試就部署新模型

## 🎓 技術原理

### 為什麼有效？

1. **持續學習（Continual Learning）**
   - 從新經驗中學習
   - 保留舊知識

2. **從人類反饋學習（Learning from Human Feedback）**
   - 用戶直接糾正錯誤
   - 用戶評價質量

3. **主動學習（Active Learning）**
   - AI 知道自己不確定什麼
   - 優先改進弱點

4. **小步快跑**
   - 每次只用少量新數據
   - 使用小學習率（1e-5）
   - 避免忘記舊知識

## 📈 預期效果

經過 3-5 次進化循環後：
- ✅ 對你的領域更專精
- ✅ 回答更符合你的偏好
- ✅ 更誠實（不懂時會說不知道）
- ✅ 錯誤率顯著降低

## 🆘 故障排除

### 問題：生成的訓練數據太少
**原因**：反饋不夠或質量標準太高  
**解決**：
```python
# 降低標準
new_samples = system.generate_training_data(
    min_confidence=0.5,  # 降低信心閾值
    min_rating=3         # 接受中等評價
)
```

### 問題：進化後效果變差
**原因**：過擬合或數據質量差  
**解決**：
1. 減少訓練輪數（epochs=1）
2. 檢查訓練數據質量
3. 恢復到舊版本模型

### 問題：模型忘記舊知識
**原因**：新數據覆蓋舊知識  
**解決**：
1. 混合舊訓練數據
2. 使用更小的學習率
3. 定期用原始數據重新校準
