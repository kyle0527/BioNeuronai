  └─ 加密貨幣市場情緒指數

  ├─ 美國 CPI/PPI 數據
  ├─ 聯準會利率決議
  ├─ 非農就業報告
  └─ 其他重要宏觀數據

□ 📰【重要】AI 新聞分析（系統自動執行）
  ├─ 選擇選項 0️⃣ 查看新聞分析
  ├─ 檢查整體情緒（正面/負面/中性）
  ├─ 查看情緒分數（-1.0 到 1.0）
  ├─ 關注重大事件標籤：
  │   ├─ 🚨 安全事件 → 立即暫停交易
  │   ├─ ⚖️ 監管相關 → 謹慎觀望
  │   ├─ 📊 ETF 相關 → 可能大波動
  │   ├─ 🤝 合作關係 → 正面訊號

  └─ 閱讀最新頭條摘要

□ 掃描加密貨幣相關新聞（補充人工判讀）
  ├─ 監管政策變動
  ├─ 交易所公告
  ├─ 主要項目更新
  └─ 安全事件報導
```

#### ✅ 系統與帳戶檢查

### 1.2 單筆交易前檢查 (Per-Trade Checklist)

**執行時機**: 每次開倉前必須完成

#### ✅ 交易信號確認 (版本 A+B 整合)
```
□ 技術分析確認
  ├─ 主要趨勢方向 (上升/下降/震盪)
  ├─ 關鍵支撐/阻力位
  ├─ 多個時間框架對齊 (15m, 1h, 4h)
  └─ 技術指標信號
      ├─ RSI 是否在合理區間
      ├─ MACD 交叉方向
      └─ 布林帶位置

□ 交易信號質量評估
  ├─ 信號強度 (1-10 分)
  ├─ 確認信號數量 (至少 2 個獨立指標)
  ├─ 與歷史成功案例相似度
  └─ AI 策略融合評分

□ 基本面檢查
  ├─ 無重大利空消息
  ├─ 無異常資金流動
  ├─ 交易量正常
  └─ 市場深度充足
```

#### ✅ 風險計算 (版本 A 1% 規則)
```
□ 計算倉位大小
  公式: 倉位大小 = (帳戶資金 × 風險百分比) / (入場價 - 止損價)
  
  例: 帳戶 $10,000, 風險 1%, BTC $50,000, 止損 $49,000
  倉位 = ($10,000 × 1%) / ($50,000 - $49,000) = $100 / $1,000 = 0.1 BTC
  
  ├─ 帳戶總資金: $__________
  ├─ 單筆風險比例: _____% (建議 1-2%)
  ├─ 最大虧損金額: $__________
  └─ 計算倉位大小: __________

□ 計算期望回報 (Expected Return)
  公式: [(勝率 × 盈利%) + (敗率 × 虧損%)]
  
  例: 勝率 60%, 盈利 4%, 敗率 40%, 虧損 2%
  期望回報 = (0.6 × 4%) + (0.4 × -2%) = 2.4% - 0.8% = 1.6%
  
  ├─ 預估勝率: _____% (基於歷史數據)
  ├─ 目標盈利: _____% (Take Profit)
  ├─ 最大虧損: _____% (Stop Loss)
  └─ 期望回報: _____% (必須 > 0.5%)

□ 盈虧比計算
  盈虧比 = 目標盈利 / 最大虧損 (建議 ≥ 1.5:1)
  
  ├─ 入場價格: $__________
  ├─ 止盈價格: $__________ (盈利: ____%)
  ├─ 止損價格: $__________ (虧損: ____%)
  └─ 盈虧比: _____:1 (必須 ≥ 1.5)
```

#### ✅ 訂單參數設定
```
□ 入場訂單
  ├─ 訂單類型: [限價/市價]
  ├─ 入場價格: $__________
  ├─ 訂單數量: __________
  └─ 槓桿倍數: _____x (建議 ≤ 10x)

□ 止損訂單 (Stop-Loss)
  ├─ 止損類型: [止損限價/止損市價]
  ├─ 觸發價格: $__________
  ├─ 執行價格: $__________
  └─ 止損理由: [技術位/固定百分比]

□ 止盈訂單 (Take-Profit)
  ├─ 分批止盈: [是/否]
  ├─ 第一目標: $__________ (_____%)
  ├─ 第二目標: $__________ (_____%)
  └─ 第三目標: $__________ (_____%)
```

#### ✅ 最終確認檢查清單
```
□ 心理狀態檢查
  ├─ 情緒穩定，無焦慮或衝動
  ├─ 未受近期虧損影響
  ├─ 精神狀態良好，注意力集中
  └─ 沒有「追單」心態

□ 交易記錄準備
  ├─ 交易日誌已準備
  ├─ 截圖工具就緒
  ├─ 交易理由已記錄
  └─ 預期目標已設定

□ 三次確認原則
  1. 再次確認信號有效性
  2. 再次確認風險可控性
  3. 再次確認符合交易計劃
  
  全部確認後才可執行交易 ✓
```

### 2.2 倉位管理流程 (Position Management)

#### ✅ 持倉期間監控 (版本 B+C 整合)
```
□ 每小時檢查清單
  ├─ 價格是否朝預期方向移動
  ├─ 止損止盈訂單狀態正常
  ├─ 未觸發異常波動警報
  └─ 保證金率充足 (> 30%)

□ 動態調整策略
  ├─ 價格達到第一目標
      └─ 考慮移動止損到成本價 (保本)
  ├─ 價格達到 50% 利潤
      └─ 考慮部分止盈鎖定利潤
  └─ 止損被觸及
      └─ 執行止損，不猶豫

□ 追蹤止損 (Trailing Stop)
  當盈利達到一定程度時啟用:
  ├─ 盈利 > 2% 時: 移動止損到成本價
  ├─ 盈利 > 4% 時: 移動止損到 +2%
  └─ 盈利 > 6% 時: 使用追蹤止損 (回撤 2%)
```

#### ✅ 倉位調整決策樹
```
[價格走勢]
    │
    ├─ 朝預期方向 (盈利中)
    │   ├─ 信號持續強化 → 考慮加倉 (謹慎)
    │   ├─ 達到目標位 → 部分止盈
    │   └─ 信號減弱 → 全部止盈
    │
    ├─ 橫向震盪
    │   ├─ 靠近止損 → 保持觀望
    │   ├─ 超時未達目標 → 考慮平倉
    │   └─ 接近止盈 → 保持耐心
    │
    └─ 反向移動 (虧損中)
        ├─ 未觸及止損 → 保持紀律，等待
        ├─ 接近止損 → 準備執行止損
        └─ 觸及止損 → 立即執行，無條件
```

## 🛡️ 風險管理檢查清單

### 5.1 資金管理規則 (版本 A 核心原則)

#### ✅ 1% 規則 (The One-Percent Rule)
```
核心原則: 單筆交易風險不超過帳戶總資金的 1%

計算公式:
───────────────────────────────────
最大風險金額 = 帳戶總資金 × 1%

例: 帳戶 $10,000
    最大風險 = $10,000 × 1% = $100

這意味著:
├─ 即使連續虧損 10 次
├─ 總虧損 = 10 × $100 = $1,000 (10%)
└─ 仍保留 90% 資金可繼續交易
───────────────────────────────────

保守型交易者: 0.5% - 1%
積極型交易者: 1% - 2%
⚠️ 絕不超過: 2%
```

#### ✅ 每日虧損限制
```
核心原則: 當日累計虧損達到限制時，停止交易

建議限制:
├─ 保守型: 2% - 3% 帳戶資金
├─ 平衡型: 3% - 5% 帳戶資金
└─ 積極型: 5% - 7% 帳戶資金

觸發後行動:
1. 立即停止所有新交易
2. 評估現有持倉
3. 必要時平倉避險
4. 休息，明天再戰

例: 帳戶 $10,000, 限制 5%
    當日虧損達 $500 → 停止交易
```

#### ✅ 總倉位限制
```
核心原則: 同時持有的總倉位不超過帳戶資金的特定比例

槓桿使用建議:
├─ 1-3x 槓桿: 可使用 50% 總資金
├─ 3-5x 槓桿: 可使用 30% 總資金
├─ 5-10x 槓桿: 可使用 20% 總資金
└─ >10x 槓桿: ⚠️ 不建議使用

保證金率維持:
├─ 最低維持: 30%
├─ 警戒線: 50%
└─ 安全線: 70%
```

### 5.2 止損策略 (Stop-Loss Strategies)

#### ✅ 技術位止損 (版本 B)
```
方法 1: 關鍵支撐/阻力位
├─ 做多: 止損設在關鍵支撐位下方 0.5-1%
└─ 做空: 止損設在關鍵阻力位上方 0.5-1%

方法 2: 移動平均線
├─ 短期: 20 EMA 或 50 EMA
├─ 中期: 100 EMA 或 200 EMA
└─ 止損設在均線下方/上方 1-2%

方法 3: ATR (Average True Range)
├─ 計算 14 期 ATR
├─ 止損距離 = 1.5 - 2 倍 ATR
└─ 適用於波動性調整

方法 4: 前期高低點
├─ 做多: 前期低點下方
├─ 做空: 前期高點上方
└─ 留 0.5-1% 緩衝空間
```

#### ✅ 固定百分比止損
```
適用場景: 快速交易，日內交易

設定標準:
├─ 低波動市場 (BTC): 2-3%
├─ 中波動市場 (ETH): 3-5%
└─ 高波動市場 (小幣): 5-10%

⚠️ 注意:
- 不要設置太緊，避免被市場噪音掃出
- 不要設置太鬆，超過風險承受能力
- 設置後不要輕易移動止損（除非往有利方向）
```

#### ✅ 時間止損
```
概念: 超過預定時間未達目標，主動平倉

適用場景:
├─ 日內交易: 持倉不過夜
├─ 短線交易: 4-8 小時內
└─ 波段交易: 1-3 天內

規則:
如果持倉超過預定時間:
├─ 無論盈虧，評估是否繼續持有
├─ 盈利 < 50%目標 → 考慮離場
└─ 虧損接近止損 → 主動止損
```

### 5.3 止盈策略 (Take-Profit Strategies)

#### ✅ 固定目標止盈
```
方法: 設定固定盈虧比

推薦比例:
├─ 保守: 1.5:1 (風險 2%, 目標 3%)
├─ 平衡: 2:1 (風險 2%, 目標 4%)
└─ 積極: 3:1 (風險 2%, 目標 6%)

例: 
入場 $50,000, 止損 $49,000 (風險 2%)
盈虧比 2:1 → 止盈 $52,000 (利潤 4%)
```

#### ✅ 分批止盈 (版本 C 推薦)
```
策略: 分多個目標位逐步止盈

標準分配:
├─ 第一目標 (1R): 平倉 30%
│   └─ 同時移動止損到成本價
├─ 第二目標 (2R): 平倉 40%
│   └─ 移動止損到 +0.5R
└─ 第三目標 (3R+): 平倉 30%
    └─ 使用追蹤止損

優點:
✓ 鎖定部分利潤，降低風險
✓ 保留倉位，捕捉大行情
✓ 心理壓力較小
```

#### ✅ 追蹤止損 (Trailing Stop)
```
方法: 隨價格有利移動而移動止損

設置方式:
1. 固定百分比追蹤
   └─ 例: 回撤 2% 時止盈

2. ATR 追蹤止損
   └─ 例: 止損跟隨價格，距離 2×ATR

3. 分階段追蹤
   ├─ 盈利 2-4%: 止損 → 成本價
   ├─ 盈利 4-6%: 止損 → +2%
   └─ 盈利 >6%: 追蹤止損 (回撤 2%)

適用場景:
✓ 強勢趨勢行情
✓ 突破後的延續走勢
✓ 捕捉大波段利潤
```

### 5.4 槓桿使用規則 (版本 C 加密貨幣專用)

#### ✅ 槓桿選擇指南
```
槓桿倍數建議:
═══════════════════════════════════
1-3x 槓桿 (保守)
├─ 適合: 新手交易者
├─ 優點: 風險可控，清算風險低
├─ 缺點: 資金效率較低
└─ 推薦交易對: BTC, ETH

3-5x 槓桿 (平衡)
├─ 適合: 有經驗交易者
├─ 優點: 平衡風險與收益
├─ 缺點: 需要密切監控
└─ 推薦交易對: 主流幣

5-10x 槓桿 (積極)
├─ 適合: 專業交易者
├─ 優點: 高資金效率
├─ 缺點: 清算風險高，需嚴格風控
└─ 推薦交易對: 高流動性幣種

>10x 槓桿 (極度危險)
├─ ⚠️ 不建議使用
├─ 清算風險極高
├─ 適合: 極短線交易（秒級/分鐘級）
└─ 必須: 止損必須極緊，隨時準備平倉
═══════════════════════════════════
```

#### ✅ 槓桿風險計算
```
清算價格計算 (簡化版):

做多清算價 = 入場價 × (1 - 1/槓桿倍數 + 維持保證金率)
做空清算價 = 入場價 × (1 + 1/槓桿倍數 - 維持保證金率)

例: BTC $50,000, 10x 槓桿, 維持保證金率 0.5%
做多清算價 = $50,000 × (1 - 0.1 + 0.005) = $45,250
做空清算價 = $50,000 × (1 + 0.1 - 0.005) = $54,750

安全距離檢查:
├─ 清算價距離 > 10%: 相對安全
├─ 清算價距離 5-10%: 需要監控
└─ 清算價距離 < 5%: ⚠️ 危險！

建議:
- 清算價應遠離技術支撐/阻力位
- 保留充足的安全邊際
- 使用隔離保證金模式（避免爆全倉）
```

### 5.5 心理風險控制 (版本 A+B 整合)

#### ✅ 情緒管理檢查清單
```
交易前自我檢查:
□ 今天身體狀態良好
□ 心情平穩，沒有焦慮或興奮
□ 沒有受到外界壓力影響
□ 不是為了「賺回虧損」而交易
□ 不是因為 FOMO (害怕錯過)而交易
□ 有充足的時間監控交易
□ 理解並接受可能的虧損

如有任何一項未滿足 → 今天不適合交易
```

#### ✅ 交易心理陷阱
```
常見錯誤與應對:

1. 復仇交易 (Revenge Trading)
   ├─ 表現: 虧損後急於翻本
   ├─ 危害: 加大風險，連續虧損
   └─ 應對: 虧損後強制休息 30 分鐘

2. 過度交易 (Over-Trading)
   ├─ 表現: 頻繁進出，交易次數過多
   ├─ 危害: 手續費侵蝕利潤，疲勞交易
   └─ 應對: 設定每日最大交易次數

3. 移動止損 (Moving Stop-Loss)
   ├─ 表現: 價格接近止損時向外移動
   ├─ 危害: 虧損擴大，風控失效
   └─ 應對: 設定止損後禁止移動（除非往有利方向）

4. 提前止盈 (Pre-mature Profit Taking)
   ├─ 表現: 小賺就跑，不給利潤奔跑空間
   ├─ 危害: 盈虧比差，難以彌補虧損
   └─ 應對: 堅持執行止盈計劃

5. 過度自信 (Overconfidence)
   ├─ 表現: 連勝後加大倉位，忽視風險
   ├─ 危害: 一次大虧抹平所有盈利
   └─ 應對: 無論輸贏，嚴格遵守倉位規則
```

#### ✅ 紀律執行原則
```
交易紀律黃金法則:

1. 計劃你的交易 (Plan Your Trade)
   ├─ 每筆交易前有明確計劃
   ├─ 記錄入場、止損、止盈
   └─ 明確交易理由

2. 交易你的計劃 (Trade Your Plan)
   ├─ 嚴格按計劃執行
   ├─ 不臨時改變主意
   └─ 接受計劃內的虧損

3. 記錄你的交易 (Record Your Trade)
   ├─ 詳細記錄每筆交易
   ├─ 定期復盤總結
   └─ 持續改進策略

紀律 > 技術 > 運氣
```

```

### 6.2 市場極端行情應對

#### ✅ 瀑布式下跌 / 暴力拉升
```
特徵:
├─ 短時間（5-15分鐘）暴漲暴跌 >5%
├─ 成交量激增
├─ 連續單邊大陰線或大陽線
└─ 爆倉清算連鎖反應

立即行動:
1. 暫停所有新開倉
2. 檢查現有持倉止損狀態
3. 如止損被穿透，手動平倉
4. 收緊在場倉位的止損
5. 等待市場穩定後再交易

事後分析:
├─ 檢查是否有重大消息
├─ 評估是否趨勢反轉
└─ 調整後續交易策略
```

#### ✅ 閃崩 / 插針 (Flash Crash / Wick)
```
特徵:
├─ 瞬間大幅下跌/上漲後快速恢復
├─ K線留下長長的影線
├─ 通常發生在流動性差的時段
└─ 可能觸發大量止損

應對策略:
1. 如止損被觸發
   └─ 評估是否重新入場
   └─ 等待價格穩定後決定

2. 如持倉未被掃出
   └─ 檢查止損是否設置過緊
   └─ 適當放寬止損空間（> 1.5× ATR）

3. 預防措施
   ├─ 不使用過高槓桿
   ├─ 止損留有緩衝空間
   └─ 使用限價止損代替市價止損
```

#### ✅ 資金費率異常
```
識別異常:
├─ 資金費率 > 0.1% (正常: 0.01%)
├─ 連續多期極端費率
└─ 多空持倉嚴重不平衡

應對措施:
1. 做多時 (支付費率)
   ├─ 考慮在費率收取前平倉
   ├─ 評估是否需要切換到現貨
   └─ 計算費率對整體收益的影響

2. 做空時 (收取費率)
   ├─ 可以適當延長持倉時間
   ├─ 享受資金費率收益
   └─ 但不要忽視價格風險

3. 一般原則
   └─ 資金費率只是參考
   └─ 價格趨勢更重要
   └─ 不要單純為了資金費率交易
```

### 7.2 SOP 整合到交易系統

#### ✅ 系統化執行
```python
# 將 SOP 編碼到交易系統中

class TradingSOPManager:
    """交易 SOP 管理器"""
    
    def pre_trade_checklist(self):
        """交易前檢查清單"""
        checklist = {
            '市場環境': self.check_market_conditions(),
            'API連接': self.check_api_status(),
            '帳戶狀態': self.check_account_status(),
            '風險參數': self.check_risk_parameters()
        }
        return all(checklist.values())
    
    def calculate_position_size(self, risk_pct=0.01):
        """計算倉位大小 (1% 規則)"""
        account_balance = self.get_account_balance()
        risk_amount = account_balance * risk_pct
        entry_price = self.get_current_price()
        stop_loss_price = self.calculate_stop_loss()
        
        position_size = risk_amount / abs(entry_price - stop_loss_price)
        return position_size
    
    def execute_trade_with_sop(self, signal):
        """按 SOP 執行交易"""
        # 步驟 1: 交易前檢查
        if not self.pre_trade_checklist():
            self.log("交易前檢查未通過，取消交易")
            return False
        
        # 步驟 2: 計算倉位
        position_size = self.calculate_position_size()
        
        # 步驟 3: 計算期望回報
        expected_return = self.calculate_expected_return(signal)
        if expected_return < 0.005:  # 低於 0.5%
            self.log("期望回報過低，取消交易")
            return False
        
        # 步驟 4: 執行交易
        order = self.place_order(position_size)
        
        # 步驟 5: 設置保護訂單
        self.set_stop_loss(order)
        self.set_take_profit(order)
        
        # 步驟 6: 記錄交易
        self.log_trade(order)
        
        return True
```
