# BioNeuronAI v2.2 發展藍圖與技術規格 (Roadmap & Spec)

> 更新日期：2026-04-24  
> 目的：確立 v2.1 穩定版之後的下一步核心開發目標，聚焦於分析模組重構、事件合約驗證，以及 RAG 系統與策略融合的最終對接。

## 🎯 核心目標：從靜態演算法邁向動態情境感知

目前 v2.1 已經完成了「動態演算法融合（Algorithmic Fusion）」的基礎建設，所有策略都能在沒有深度學習模型直接介入的情況下，依靠啟發式權重穩定運作（AI 模型目前處於待命 Standby 狀態）。

v2.2 的核心目標是讓系統具備**「情境感知（Context-Awareness）」**，特別是針對突發新聞與市場事件，能夠精準計算事件的持續時間與影響力，並動態調整策略權重。

---

## 🏗️ Phase 1: 分析模組重構與事件合約 (Event Contract)

這是目前最核心的待辦技術債（原 T4/T5）。

### 1.1 `news/analyzer.py` 拆分
目前的 `CryptoNewsAnalyzer` 過於龐大（混合了抓取、過濾、情緒分析）。
*   **目標**：將其拆分為 `NewsFetcher` (資料獲取，移至 `data/` 層)、`NewsProcessor` (去重與正規化)、`SentimentEngine` (情緒計算與關鍵字比對)。
*   **效益**：提升可測試性，並符合 `data/` 負責外部 API 的架構原則。

### 1.2 事件合約驗證框架 (NewsEventContract)
目前的情緒分數缺乏「時間維度」，導致利多新聞的影響力可能被無限期延長。
*   **目標**：實作 `NewsEventContract`。
*   **規格**：
    *   每個重大新聞生成一個 Contract，包含：`event_type`, `urgency`, `initial_impact`, `decay_rate`, `expires_at`。
    *   **自適應衰減**：新聞影響力隨時間呈指數或線性衰減。
    *   **驗證閉環**：在事件到期後，系統自動回顧該新聞期間的真實價格變化（PnL），將結果標註為高品質的訓練資料（供未來模型使用）。

---

## 🧠 Phase 2: RAG 知識庫對接策略融合

目前的 RAG 系統（檢索增強生成）已經能將盤後日報與新聞存入 FAISS 向量資料庫，並在 `pretrade` 階段進行風險提示，但尚未與即時策略權重掛鉤。

### 2.1 充實 `EventContext`
*   **目標**：在 `TradingPlanController` 呼叫策略前，主動向 `InternalKnowledgeBase` 查詢當前交易對與市場狀態的歷史相似事件。
*   **實作**：將檢索到的歷史勝率與交易規則，封裝進 `EventContext` 中。

### 2.2 策略融合權重調整
*   **目標**：`AIStrategyFusion` 接收到包含 RAG 檢索結果的 `EventContext` 後，動態調整子策略權重。
*   **情境範例**：若 RAG 檢索發現「過去 3 次類似的 BTC 突破事件都遭遇假突破」，則融合引擎自動調降 `BreakoutStrategy` 的權重，並提高 `MeanReversionStrategy` 的權重。

---

## 🚀 Phase 3: 模型重啟計畫 (Meta-Learner / RL Agent)

在 Phase 1 與 Phase 2 穩定，且系統累積了足夠多帶有 `NewsEventContract` 驗證結果的高品質資料後，啟動深度學習模型。

*   **目標**：將目前處於 Standby 的 `my_100m_model.pth` 轉化為 Meta-Learner。
*   **作用**：取代目前 `AIStrategyFusion` 中的靜態啟發式權重分配，改由神經網路直接根據 1024 維特徵與 EventContext，預測各子策略的最佳動態權重組合。

---

## 📌 執行優先順序建議

1.  **優先實作 `NewsEventContract` (Phase 1.2)**：這是收集高質量時間維度訓練資料的前提。
2.  **重構 `analyzer.py` (Phase 1.1)**：在實作合約的同時，順便梳理程式碼。
3.  **接通 RAG 與 EventContext (Phase 2)**：讓系統具備歷史記憶。
4.  **模型重啟 (Phase 3)**：最後的長期目標。
